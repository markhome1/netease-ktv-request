# 网易云官方 OpenAPI 路线

## 结论

当前项目更适合接入“网易云官方 OpenAPI + 本地主机播放器”模式，而不是尝试直接远程控制网易云 PC 客户端。

原因：

- 官方开放平台明确提供了用户登录、文字搜索、查询歌单及歌曲、获取播放地址、收藏&创建、跨端续播等 API 分类。
- 官方文档中没有明显暴露“远程控制网易云桌面客户端播放队列”的通用 Web 开发接口。
- 当前局域网点歌系统最核心的是“扫码收歌 + 主机统一播放”，这正好更适合用官方 API 提供内容，用自己的主机播放器执行播放。

## 对当前项目的能力映射

### 1. 手机扫码点歌

- 当前项目已具备，保留现有实现。
- 用户提交：歌名、歌手、网易云链接、备注。

### 2. 官方搜索能力

- 文档导航中存在“文字搜索 API”。
- 适合替换当前手填歌曲模式，改成手机端直接搜索网易云歌曲并选择。

### 3. 官方歌单查询能力

- 文档导航中存在“查询歌单及歌曲 API”。
- 适合用于：
  - 读取你的目标歌单
  - 校验歌曲是否已在歌单里
  - 做歌单去重与回显

### 4. 官方播放地址能力

- 文档导航中存在“获取播放地址 API”。
- 文档说明会员身份会影响返回结果：会员可得到可播放链接，非会员可能只有试听片段或无链接。
- 对本项目最关键：你是 SVIP，理论上更适合让主机后端去拿官方播放地址后直接播放。

### 5. 官方歌单写入能力

- 文档导航中存在“收藏&创建 API”。
- 这很可能对应“创建歌单、收藏、向歌单添加歌曲”等能力。
- 对本项目的意义：待审核通过后，把歌曲同步到你的网易云目标歌单。

### 6. 用户登录能力

- 文档导航中存在“用户登录 API”。
- 这意味着官方路线需要真实登录授权，而不是简单 cookie 模式。
- 项目后续应增加授权回调、令牌存储、刷新策略。

### 7. 跨端续播能力

- 文档导航中存在“跨端续播 API”。
- 这类能力更像“播放状态同步”而不是“浏览器远程控制网易云桌面客户端”。
- 可以研究，但不应作为当前主播放方案的基础依赖。

## 当前工程应如何切换

### 第一阶段：官方模式骨架

- 保留现有扫码、审核、队列、主机播放逻辑。
- 将当前“第三方网易云 API 兼容模式”抽象成 provider。
- 新增 provider: official-openapi。
- 配置项改为：appId、openapiBaseUrl、privateKey、publicKeyFingerprint、loginMode、userToken 等。

### 第二阶段：先打通读取链路

优先实现：

1. 用户登录
2. 歌曲搜索
3. 获取播放地址
4. 主机本地播放

这四步打通后，系统已经能作为“官方内容源 + 局域网点歌机”运行。

### 第三阶段：再打通歌单写入

后续再接：

1. 创建/选择目标歌单
2. 审核通过后写入歌单
3. 写入失败重试
4. 歌单内去重

## 当前阻塞项

以下信息仍未从当前抓取结果中拿到精确字段级定义，因此暂时不直接落真实官方请求代码：

- 官方 RSA 加密与 sign 拼接规则的字段级示例
- 公共参数拼装与签名落地方式
- 把匿名登录、二维码轮询、refresh token 续期串成真实服务端流程
- 把已确认的搜索、播放地址、歌单写入、歌单读取能力真正切到 official-openapi provider

当前项目可以继续运行第三方兼容模式；但官方模式所需字段现在已经基本补齐，后续主要是请求层实现，不再是文档缺口。

## 推荐决策

- 主机播放：继续用你自己的主机播放器，不依赖网易云 PC 客户端。
- 网易云内容源：逐步切到官方 OpenAPI。
- PC 客户端控制：不作为当前版本目标。
